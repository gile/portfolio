case study

As part of my contribution to a BI dashboard solution, I was responsible for the UX design of a new web-based interface.
My approch was to first understand what the basic flows are, where a flow is a set of actions a user ,must take in order to complete a specific goal.

For each flow I had prepared a series of mockups used to illustrate to the R&D team, Marketing and Graphic Designers how the product should look and behave.

Create Widget

One of the first tasks a user will encounter is the need to create a widget (e.g. graph, map, or KPI). I sought to make the process less confusing by integrating hints which display the first time a widget is created.

When creating a widget, a user must first define the datasource he wants to use, and what type of widget will be created.
This is illustrated by the big digits, so that he is aware of these two steps.

Early on, it was decided that there should be a clear separation between the data aspects of the widget (on the left), and the style aspects on the right.

A major challenge was the design of the data-browser - a pop-up used to select a data source.
This browser must be compact and easy to use, yet display a large amount of information.
To help the user find his data quickly, an auto-complete text box was used.

A map widget after data and styling attributes have been added.

Move Widget

Moving widgets around the dashboard is accomplished in two step.
First, the user enters "Edit Mode" by clicking a button in the top menu. A different color scheme is then applied.

Second, the widgets are positioned using a drag-and-drop approach.
As the user hovers near a legal position for the widget, it is highlighted.

In Edit Mode, widgets can either be re-positioned or resized.

The widget positions are based on a column layout.
A vertical marker notifies the user that the current placement will breate a new column.

When the widget is placed in this position, the new column is created.
